# AI_CHANGELOG.md

## 2026-04-14
- **Change**: Initialized Master PRD and core Godot `project/` architecture.
- **Reasoning**: To establish a solid Feature/Entity-based directory layout for Godot 4.x before writing engine logic. Removed initial web boilerplate upon learning this is a standalone PC game. 

- **Change**: Created `project.godot` manifest and core singletons (`SaveManager.gd`, `TimeEngine.gd`).
- **Reasoning**: Establishing the foundational simulation logic (Offline Time Engine) and defining pixel-perfect 640x360 project settings. Enables the user to "Import" the project into Godot immediately.

- **Change**: Wired `HUD.gd` to `TimeEngine.gd` signals and added `OfflineLabel`.
- **Reasoning**: Completing the first gameplay feedback loop. The game now successfully communicates its unique "offline simulation" mechanic to the user via the UI.

- **Change**: Implemented `Iridescent.gdshader` and `GhostTrail.gd` logic for Odin.
- **Reasoning**: Establishing the core "liquid-rainbow" visual aesthetic. Movement now feels "divine" and high-polish, meeting the art goals defined in the Master PRD.

- **Change**: Added "Go to Sleep" button, fade transition, and world-position persistence.
- **Reasoning**: Completing the core gameplay loop. Players can now save their exact location and inventory state, ensuring a seamless experience when they return after the "All-Sleep."

- **Change**: Implemented `HarvestableNode.gd` and `DivineSpark.tscn` particle effects.
- **Reasoning**: Adding the first layer of world interaction. Odin can now "harvest" resources (Star-Iron) that trigger holographic visual feedback and update the persistent inventory system.

- **Change**: Created `Proxy.tscn` chassis and context-overlay logic.
- **Reasoning**: Establishing the automation foundation. Proxies can now be deployed and visually branded (Dwarven/Elven) to prepare for the upcoming Alignment Bezel programming system.

- **Change**: Implemented the `AlignmentBezel` 3-dial mini-game and wired it to the Proxy system.
- **Reasoning**: Completing the core "Automation/Programming" loop. Players can now physically "tune" their proxies via a high-end mechanical UI, with performance-based feedback loops (Perfect vs. Slipped states).

- **Change**: Implemented Proxy Simulation logic for offline harvesting.
- **Reasoning**: Completing the "Idle/Cozy" core loop. Programmed proxies now automatically harvest resources based on the real-world time elapsed between game sessions.

- **Change**: Created `Nidavellir.tscn` level and `BifrostTransit.gd` scene teleport system.
- **Reasoning**: Expanding the game world. Players can now travel between the lush Asgard hub and the industrial Nidavellir forge, establishing the foundation for realm-specific resources and visuals.

- **Change**: Implemented the `Proxy Squad` system (Pick-up E, Deploy Q).
- **Reasoning**: Enabling multi-realm automation.
